# 视频内容智能打标系统 - 毕业设计分析报告

## 一、项目概述

### 1.1 项目名称
视频内容智能打标系统（Video Content Intelligent Tagging System）

### 1.2 技术栈
- **开发语言**: Python 3.11.7
- **UI 框架**: Tkinter (原生 GUI)
- **图像处理**: OpenCV, PIL/Pillow
- **AI 模型**: Qwen-VL (通过 Ollama/vLLM API 调用)
- **配置文件**: INI 格式 (configparser)

### 1.3 项目结构
```
video_mark_tool/视频打标器/code/
├── video_tagger.py      # 主程序入口，UI 初始化
├── ui_events.py         # UI 事件处理模块

├── ai_features.py       # AI 功能模块
├── presets.py           # 预设管理模块
├── utils.py             # 工具函数模块
└── config.ini           # 配置文件
```

## 四、关键技术实现原理

### 4.1 视频帧预处理机制

**目的**: 解决长视频加载慢、内存占用高的问题

**算法流程**:
```python
# 计算采样间隔
if original_fps > target_fps:
    frame_interval = original_fps / target_fps
else:
    frame_interval = 1

# 均匀采样
for i in range(total_frames):
    if i % frame_interval == 0:
        extract_frame(i)
```

**参数配置**:
- 目标 FPS: 16 (可配置)
- 目标高度：720p (保持宽高比)
- 最大采样帧数：15 帧/AI 处理请求

### 4.2 AI 标签生成流程

**核心步骤**:
1. **帧提取**: 从标记区间均匀采样最多 15 帧
2. **图像编码**: PIL 缩放到 720x720 → PNG → Base64 Data URL
3. **API 调用**: 构造多模态消息体发送给 Ollama/vLLM
4. **结果验证**: 
   - 长度检查 (>50 字符)
   - 敏感词过滤 (正则匹配 + 重试机制)
   - 思考标签移除 (`` 分割)

**API 消息格式**:
```json
{
  "model": "qwen3-vl:9b",
  "messages": [{
    "role": "user",
    "content": [
      {"type": "text", "text": "详细描述视频内容..."},
      {"type": "image_url", "image_url": {"url": "data:image/png;base64,..."}},
      ...
    ]
  }],
  "max_tokens": 16384,
  "temperature": 0.3,
  "top_p": 0.95
}
```

### 4.3 时间轴可视化渲染

**Canvas 绘制顺序**:
1. 灰色背景矩形 (整个时间轴)
2. 彩色标记矩形 (不同标签，带 stipple 半透明效果)
3. 红色垂直线 (当前帧位置)
4. 绿色/紫色线 (开始/结束帧标记)
5. 时间刻度文字

**坐标转换**:
```python
x = (frame_index / total_frames) * canvas_width
```

## 五、性能优化策略

### 5.1 内存管理
- **帧缓存**: 所有预处理帧存储在内存中，避免重复解码
- **图像引用**: 使用 `photo.image = photo` 防止 GC 回收
- **窗口清理**: 关闭加载窗口时显式销毁

### 5.2 线程分离
- **AI 处理**: 独立线程执行，避免阻塞 UI
- **批量任务**: 后台线程运行，主线程更新进度
- **UI 更新**: `root.after()` 确保线程安全

### 5.3 响应式设计
- **拖拽面板**: PanedWindow 支持左右宽度调节
- **字体缩放**: 动态调整全局字体大小
- **窗口居中**: 根据屏幕尺寸自动计算位置
